The File Structure

  • CHANGELOG.md
  • config.yml
  • /etc/
  • gulpfile.babel.js
  • /node_modules/
  • package.json
  • readme.md
  • /src/
    • /assets/

      where you can add other stylesheets, images, javascript files, and more.

    • /data/

      here you can add .yml files and .json files w/ some data.

    • /layouts/

      Similar to a Master Page, contains just 1 file, here you will include the header, footr, and other sections that should appear on all pages (links to stylesheets and javascript files too)

    • /pages/

      one file for each page in the website
      (exp) - about.html, gallery.html, clients.html

    • /partials/

      HTML code that you want to use on different pages (but not all pages), think widgets.

    • /styleguide/

Sass Mixins

  • If you find yourself writing the same CSS over and over, or you have a complex piece of CSS you want to reuse, then MIXINS is where it's at!
  • A mixin let's you make groups of CSS declarations that you want to resuse throughout your site
  • You can even pass values, similar to functions in programming < to make smarter mixins.

How to use

@include my_font() {font-family: 'Helvetic', sans-serif}

Then you use it: p { @include my_font(); }

You can also pass parameters to mixins.

@include message($color) { border: 1px solid $color };

Use it: .error { @include message(red); }

  • You don't have to create all the mixins, Foundation already provides a whole set of them.
  • Mixins for media queries, display property, margin, and padding are included.
  • If one doesn't exist you can create your own!

Rem-Calc

  • Foundation uses Rems as the measurement unit for Font Sizes, Margins, Paddings
  • How to work w/ them:
    • rem is relative to the HTML document and is a division between your base font size (that you set on the HTML tag in your stylesheet) and your desired size.
    • p { font-size: rem-calc(20); } makes p { font-size: 1.125rem; }
    • Foundation made the rem-calc function to make life easier for us.
    • Shorthand Margin or Padding
    • margin: rem-calc(20, 30);
